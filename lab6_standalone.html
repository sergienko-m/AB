Лабораторна робота 6

Розробка прототипу розроблюваної системи

Мета: навчитись проектувати екранні форми розроблюваної системи.


Програмна система: GitHub

1. Тема та мета

Тема роботи: розробка прототипу екранних форм платформи GitHub — веб-сервісу для хостингу та спільної розробки програмного забезпечення на базі системи контролю версій Git.

Розроблено 7 прототипів екранних форм системи:

  • Головна сторінка репозиторію — перегляд коду, файлів та README;
  • Issues — список задач та звітів про помилки;
  • Pull Requests — список запитів на злиття коду;
  • Деталі Pull Request — code review та CI-перевірки;
  • GitHub Actions — CI/CD pipeline та запуски workflow;
  • Налаштування репозиторію — конфігурація та адміністрування;
  • Профіль користувача — статистика та activity graph.

2. Прототипи екранних форм

Рис. 1. Прототип екранної форми «Головна сторінка репозиторію»

Головна сторінка репозиторію містить навігаційну панель із пошуком, підменю розділів (Code, Issues, Pull Requests, Actions, Settings), заголовок репозиторію зі статистикою (зірки, форки, watching), дерево файлів з останніми комітами, вміст README-файлу та бокову панель з описом, темами, мовами і контрибьюторами.

Рис. 2. Прототип екранної форми «Issues (Задачі)»

Форма «Issues» відображає список задач та звітів про помилки проєкту. Кожен рядок містить статус (відкрито/закрито), заголовок, мітки (bug, enhancement, security, documentation), автора, дату створення та кількість коментарів. Доступні фільтри за міткою та виконавцем і кнопка створення нового issue.

Рис. 3. Прототип екранної форми «Pull Requests»

Форма «Pull Requests» показує список запитів на злиття коду. Для кожного PR відображається: статус (open/draft), заголовок, мітки, гілки (джерело → ціль), автор та статус CI-перевірок (✅ passed / ❌ failed / ⏳ pending). Підтримується фільтрація та сортування.

Рис. 4. Прототип екранної форми «Деталі Pull Request»

Форма деталей PR містить заголовок та статус, вкладки Conversation / Commits / Files changed, стрічку коментарів code review, блок статусів CI-перевірок (build, test, security scan) та кнопку «Merge pull request». Права панель показує рев'юерів, виконавців, мітки та мілстоун.

Рис. 5. Прототип екранної форми «GitHub Actions (CI/CD)»

Форма «GitHub Actions» відображає CI/CD pipeline репозиторію. Ліва панель містить список workflows (CI Pipeline, Deploy, Security Scan, CodeQL). Права область показує список запусків із статусами (success / failed / running), назвою коміту, гілкою та часом виконання. Доступна кнопка ручного запуску та фільтри.

Рис. 6. Прототип екранної форми «Налаштування репозиторію»

Форма «Settings» дозволяє конфігурувати параметри репозиторію: назву, опис та URL; перемикачі функцій (Issues, Wikis, Projects, Discussions); стратегії злиття PR (merge/squash/rebase) та автовидалення гілок. Розділ «Danger Zone» містить кнопки зміни visibility та видалення репозиторію.

Рис. 7. Прототип екранної форми «Профіль користувача»

Профіль користувача містить аватар, ім'я та логін, біографію, статистику (followers/following), місцезнаходження та посилання. Права частина відображає закріплені репозиторії (до 6) та Contribution Activity Graph — теплову карту активності за рік у вигляді сітки 52×7 комірок.

3. Висновок

У лабораторній роботі розроблено 7 прототипів екранних форм платформи GitHub, що охоплюють повний основний workflow системи: від перегляду репозиторію та управління задачами до CI/CD автоматизації та адміністрування.

Прототипування дозволило наочно представити інтерфейс системи до початку її реалізації, виявити потенційні проблеми у навігації та продемонструвати відповідність нефункціональним вимогам щодо зручності використання. Розроблені прототипи можуть слугувати технічним завданням для розробників та основою для обговорення із замовником.
